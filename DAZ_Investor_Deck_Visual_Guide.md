# DAZ Investor Deck - Visual Design Guide

## Design System Overview

### Brand Identity
- **Style:** High-tech, minimalist, data-driven, authoritative
- **Aesthetic:** Clean, modern, professional
- **Tone:** Confident, innovative, solution-oriented

### Color Palette

**Primary Colors:**
- **Deep Blue:** #0A2540 or #1E3A5F (primary brand color, trust, technology)
- **Electric Blue:** #0066FF or #0052CC (accent, highlights, CTAs)
- **Charcoal Gray:** #2C3E50 or #34495E (text, secondary elements)
- **Light Gray:** #ECF0F1 or #F5F7FA (backgrounds, subtle elements)

**Accent Colors:**
- **Success Green:** #27AE60 (positive metrics, growth)
- **Warning Orange:** #F39C12 (cautions, attention)
- **Error Red:** #E74C3C (risks, problems - use sparingly)

**Neutral:**
- **White:** #FFFFFF (primary background, contrast)
- **Black:** #000000 (text, strong contrast)

### Typography

**Primary Font (Headings):**
- **Inter** (preferred) or **Helvetica Neue** or **SF Pro Display**
- Weight: Bold (700) for main headlines, Semi-bold (600) for subheadings
- Style: Clean, modern sans-serif

**Secondary Font (Body):**
- **Inter** or **Helvetica Neue** or **SF Pro Text**
- Weight: Regular (400) for body text, Medium (500) for emphasis
- Size: Minimum 18pt for readability in presentations

**Font Hierarchy:**
- **Slide Title:** 48-60pt, Bold
- **Section Headers:** 32-36pt, Semi-bold
- **Body Text:** 18-24pt, Regular
- **Captions/Notes:** 14-16pt, Regular

### Layout Principles

**Grid System:**
- Use consistent margins (10-15% of slide width)
- Maintain visual hierarchy with clear spacing
- Follow rule of thirds for key elements
- Maximum 3-4 main elements per slide

**White Space:**
- Generous white space for minimalist aesthetic
- Don't overcrowd slides
- Let key elements breathe

**Alignment:**
- Left-align text (or center for title slides)
- Consistent alignment throughout deck
- Use guides for visual consistency

## Slide-by-Slide Visual Specifications

### SLIDE 1: Title Slide

**Layout:**
- Centered alignment
- DAZ logo at top (large, prominent)
- Title and subtitle stacked vertically, centered
- Tagline below subtitle, slightly smaller
- Contact info at bottom (smaller font)

**Visual Elements:**
- **Background:** Subtle circuit board/motherboard pattern
  - Very low opacity (5-10%)
  - Monochrome (gray/blue tones)
  - Geometric, tech-inspired pattern
- **Logo:** DAZ circular logo with stick figures
  - Positioned prominently at top or center-left
  - Size: 20-25% of slide height
- **Color Scheme:** Deep blue and white primary

**Design Notes:**
- Maximum impact, minimal elements
- Let the logo and tagline speak
- Background pattern should be subtle, not distracting

---

### SLIDE 2: The Problem

**Layout:**
- Left-aligned content
- Headline at top (large, bold)
- Key points in bullet format or numbered list
- Market data in highlighted boxes or callouts

**Visual Elements:**
- **Primary Visual:** Bar chart or pie chart showing time allocation
  - 99% manual work (large bar in red/orange)
  - 1% decision-making (tiny bar in green)
  - Clear visual contrast
- **Secondary Visual:** Comparison diagram
  - Left side: Fragmented, chaotic process (red/orange)
  - Right side: Integrated system (blue/green)
  - Arrow showing transformation
- **Market Data Boxes:**
  - Highlighted callout boxes for market size numbers
  - Use accent color (electric blue) for emphasis
  - Include source citations in smaller font

**Design Notes:**
- Use red/orange tones to emphasize pain/problem
- Make the 99% statistic visually dramatic
- Keep text concise, let visuals tell the story

---

### SLIDE 3: The Solution

**Layout:**
- Headline at top with GPS metaphor
- Five agent icons arranged in a grid or circular pattern
- Value proposition highlighted below

**Visual Elements:**
- **Primary Visual:** GPS Navigation Metaphor
  - Left: Old paper map (faded, torn, confusing)
  - Right: Modern GPS screen (clear, precise, blue)
  - Arrow showing transformation
- **Agent Diagram:**
  - Five circular icons representing each agent
  - Connected by lines showing coordination
  - Each icon with distinct color but unified style
  - Icons: Finance (green), Planning (blue), Legal (red), Marketing (purple), PM (orange)
- **Before/After Comparison:**
  - Top: Fragmented manual process (messy, disconnected)
  - Bottom: Unified AGI platform (clean, connected, organized)

**Design Notes:**
- Use blue tones to represent solution/technology
- Make the transformation visually clear
- Icons should be modern, minimalist, tech-inspired

---

### SLIDE 4: Product (The MVP)

**Layout:**
- Headline at top
- Six modules displayed in a grid (2x3 or 3x2)
- Flow diagram showing user journey

**Visual Elements:**
- **Module Grid:**
  - Six equal-sized boxes/cards
  - Each module with:
    - Icon (minimalist, tech-inspired)
    - Module name
    - Brief description (1-2 lines)
  - Consistent styling across all modules
  - Subtle hover/interaction effect suggestion
- **Flow Diagram:**
  - Left: "User Input" (input icon)
  - Center: "6 Modules" (connected boxes)
  - Right: "Output" (output/document icon)
  - Arrows showing flow direction
  - Clean, linear flow

**Design Notes:**
- Use consistent icon style (outline or filled, but unified)
- Module cards should feel clickable/interactive
- Flow should be intuitive and easy to follow

---

### SLIDE 5: Technology

**Layout:**
- Headline at top
- Architecture diagram as primary visual
- Key capabilities listed alongside or below

**Visual Elements:**
- **System Architecture Diagram:**
  - Multi-agent system visualization
  - Central "Orchestrator" or "Core" in center
  - Five agents arranged around it (circular or star pattern)
  - Data sources at bottom
  - Clean lines, modern tech aesthetic
  - Use blue tones for technology elements
- **Performance Comparison:**
  - Side-by-side comparison
  - Left: "Traditional: 2-4 weeks" (long timeline, red)
  - Right: "DAZ: Minutes" (short timeline, green)
  - Visual time representation (clocks, calendars, or bars)
- **Technology Stack Icons:**
  - Small icons/logos for key technologies
  - Arranged horizontally or in a grid
  - Subtle, not overwhelming

**Design Notes:**
- Architecture diagram should look professional and technical
- Use blue/tech colors throughout
- Keep it clean and understandable, not overly complex

### SLIDE 6: Market Opportunity

**Layout:**
- Headline at top
- Large market size chart as primary visual
- Market segments listed below or to the side

**Visual Elements:**
- **Market Growth Chart:**
  - Line chart or bar chart showing growth
  - X-axis: Years (2024-2034)
  - Y-axis: Market size ($B)
  - PropTech line in electric blue
  - Technology in Real Estate line in deep blue
  - Clear labels and data points
  - Projected growth shown with dotted line
- **Market Segmentation:**
  - Pie chart or horizontal bar chart
  - Four segments: Residential, Commercial, Investors, Agencies
  - Each with distinct color
  - Percentages or relative sizes shown
- **TAM/SAM/SOM Diagram:**
  - Three concentric circles (if data available)
  - TAM (outer, largest)
  - SAM (middle)
  - SOM (inner, smallest)
  - Labeled clearly

**Design Notes:**
- Use growth charts to show opportunity
- Make numbers prominent and easy to read
- Use blue/green tones for positive growth
- Include source citations

### SLIDE 7: Business Model

**Layout:**
- Headline at top
- Three-column pricing comparison table
- Additional revenue streams below

**Visual Elements:**
- **Pricing Table:**
  - Three columns: Tier 1, Tier 2, Tier 3
  - Each column with:
    - Tier name and icon
    - Key features (bullet list)
    - Target audience
    - Visual indicator of value (size, color intensity)
  - Tier 3 should appear most premium (larger, more prominent)
  - Use gradient or color intensity to show progression
- **Revenue Model Diagram:**
  - Central "DAZ Platform" circle
  - Revenue streams branching out:
    - Subscriptions (primary, largest)
    - Per-project fees
    - Premium support
    - Custom integrations
  - Size of branches indicates relative importance
- **Unit Economics:**
  - Simple metrics display
  - CAC, LTV, Gross Margin
  - Use icons and numbers
  - Green for positive metrics

**Design Notes:**
- Make pricing tiers visually distinct
- Show clear value progression
- Use professional table/chart styling
- Keep numbers and metrics prominent

### SLIDE 8: Roadmap

**Layout:**
- Headline at top
- Timeline/Gantt chart as primary visual
- Key milestones highlighted

**Visual Elements:**
- **Timeline Chart:**
  - Horizontal timeline from Q2 2026 to 2027
  - Three phases shown as colored bars:
    - Phase 1: Blue (Q2 2026)
    - Phase 2: Electric Blue (Q4 2026)
    - Phase 3: Deep Blue (2027)
  - Milestones marked with icons or markers
  - Clear labels for each phase
- **Phase Details:**
  - Each phase with:
    - Phase name and timeframe
    - Key deliverables (checkmarks or bullets)
    - Milestones (star icons or highlights)
  - Arranged vertically or in columns
- **Growth Trajectory:**
  - Small line chart showing customer/revenue growth
  - Overlaid on timeline or separate
  - Shows progression from MVP to scale

**Design Notes:**
- Timeline should be clear and easy to follow
- Use color coding for phases
- Make milestones visually distinct
- Show progression and growth

### SLIDE 9: The Team

**Layout:**
- Headline at top
- Team members in a row or grid
- Each with photo, name, title, key credentials

**Visual Elements:**
- **Team Member Cards:**
  - Three cards (or more if advisors included)
  - Each card with:
    - Professional photo (circular or rounded square)
    - Name (bold, large)
    - Title/role (smaller, colored)
    - 2-3 key credentials or achievements (bullet points)
  - Consistent styling across all cards
  - Equal sizing and spacing
- **Expertise Areas:**
  - Small icons or tags showing expertise
  - Real Estate, Law, AI/Technology
  - Color-coded or icon-based
- **Team Structure:**
  - Optional: Organizational chart
  - Founders at top
  - Advisors below (if applicable)
  - Clean, hierarchical layout

**Design Notes:**
- Professional photos (if available) or placeholder avatars
- Keep credentials concise and impactful
- Use consistent card design
- Show diversity of expertise

### SLIDE 10: Closing / The Ask

**Layout:**
- Tagline reinforcement at top
- Use of funds breakdown
- Growth projections
- Clear ask

**Visual Elements:**
- **Use of Funds Pie Chart:**
  - Four segments:
    - Tech Development (40-50%) - Largest, blue
    - Data Partnerships (20-25%) - Medium, electric blue
    - Hiring (15-20%) - Medium, deep blue
    - Go-to-Market (10-15%) - Smallest, light blue
  - Clear labels with percentages
  - Total amount prominently displayed
- **Growth Projection Chart:**
  - Line chart or bar chart
  - X-axis: Years (Year 1, 2, 3)
  - Y-axis: ARR ($M) or Customers
  - Shows growth trajectory
  - Use green/blue for positive growth
- **Key Metrics:**
  - Large numbers for:
    - Funding ask ($3-5M)
    - Year 2-3 ARR targets
    - Customer targets
  - Displayed prominently
  - Use accent colors for emphasis

**Design Notes:**
- Make the ask clear and prominent
- Show confidence with growth projections
- Use of funds should be transparent
- End on a strong, confident note

## Visual Elements Library

### Icons

**Recommended Icon Style:**
- **Outline style** (preferred) or **Filled style** (for emphasis)
- **Consistent stroke width** (2px)
- **Rounded corners** (modern, friendly)
- **Minimalist design** (simple, recognizable)

**Key Icons Needed:**
- Finance/Money (dollar sign, chart)
- Planning (map, blueprint)
- Legal (scale, document)
- Marketing (megaphone, target)
- Project Management (checklist, timeline)
- Technology/AI (circuit, brain, chip)
- Growth (arrow up, chart)
- Team (people, user)
- GPS/Navigation (location pin, map)

**Icon Sources:**
- Heroicons
- Lucide Icons
- Feather Icons
- Custom-designed for brand consistency

### Charts & Graphs

**Chart Types:**
- **Bar Charts:** For comparisons (time waste, market size)
- **Line Charts:** For growth trends (market growth, revenue)
- **Pie Charts:** For composition (use of funds, market segments)
- **Timeline/Gantt:** For roadmaps and phases
- **Flow Diagrams:** For processes and user journeys

**Chart Styling:**
- Clean, minimalist design
- Consistent color palette
- Clear labels and legends
- Professional data visualization
- Avoid 3D effects or excessive decoration

### Data Visualization Principles

1. **Clarity First:** Charts should be immediately understandable
2. **Consistent Colors:** Use brand colors throughout
3. **Proper Scaling:** Don't mislead with scale manipulation
4. **Clear Labels:** All axes and data points labeled
5. **Source Citations:** Include data sources where applicable

## Presentation Tips

### Slide Transitions
- Use subtle transitions (fade, slide)
- Avoid flashy or distracting effects
- Maintain professional tone

### Animation (if used)
- **Entrance animations:** Subtle fade-in or slide-in
- **Emphasis:** Use sparingly for key points
- **Timing:** Keep animations quick and purposeful
- **Consistency:** Use same animation style throughout

### Speaker Notes
- Each slide should have supporting speaker notes
- Key talking points for each visual
- Data points to emphasize
- Transitions between slides

### Print/PDF Version
- Ensure slides are readable in grayscale
- High contrast for text
- All information visible without animation
- Include slide numbers

## Technical Specifications

- **Standard:** 16:9 aspect ratio (1920x1080px)

- **Backup:** High-resolution images (PNG/JPG)

- **Minimum:** 300 DPI for print
- **Display:** 1920x1080 for presentations
- **Logo:** Vector format (SVG) preferred

### Color Mode
- **RGB** for digital presentations
- **CMYK** for print materials (if needed)
